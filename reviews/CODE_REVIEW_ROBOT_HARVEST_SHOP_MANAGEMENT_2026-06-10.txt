Date: 2026-06-10

Scope

1. app/api/robot.py

New route: POST /robot/harvest
- Accepts JSON from TurtleBot or robot control code.
- Required payload:
  {
    "size_class": "대",
    "quality_grade": "상"
  }
- Optional payload:
  {
    "product_name": "사과"
  }
- Calls app.db.sales.record_harvest_event().
- Returns event id, product name, size, quality grade, added estimated weight, current base inventory, and current available inventory.

Review notes
- Good MVP shape: robot only sends the classification result, while server applies fixed weight rules.
- Validation currently allows only known size and quality values through the DB function.
- API raises 400 on invalid payload values.
- If multiple robots are added later, robot_id/event_id can be added without changing the current inventory flow.

3. app/db/schema.sql

New table: apple_inventory
- Stores current base inventory by product, size, and quality grade.
- Important columns:
  product_name
  size_class
  grade
  available_kg
  reserved_kg
  package_unit
  sales_channel
  updated_at
- Unique key prevents duplicate inventory rows for the same product/size/grade.

New table: harvest_events
- Stores every robot harvest event.
- Important columns:
  product_name
  size_class
  quality_grade
  estimated_weight_kg
  created_at

New table: app_settings
- Stores configurable app settings such as shop_page_title.

Review notes
- created_at is generated by DB, so TurtleBot does not need to send a timestamp.
- Dummy inventory is now used as initial inventory, while later robot events are logged separately.

4. app/db/sales.py

New functions
- decimal_to_float(value)
  Normalizes Decimal DB values to rounded float values.

- get_sample_product(product_name, size_class, grade)
  Retrieves static product metadata such as package unit, sales channel, and fallback price.

- ensure_inventory_tables()
  Creates apple_inventory and harvest_events if missing.
  Seeds apple_inventory with the existing dummy product data using INSERT IGNORE.

- list_inventory_products()
  Reads inventory rows from DB and combines them with sample metadata.

- record_harvest_event(size_class, quality_grade, product_name="사과")
  Core robot integration function.
  Saves harvest event.
  Adds fixed estimated weight to apple_inventory.
  Creates notification.
  Returns updated inventory values.

Changed functions
- get_product_template()
  Now reads inventory from DB instead of relying only on SAMPLE_PRODUCTS.

- get_available_kg()
  Now supports decimal kg values.

- list_products()
  Now uses apple_inventory as the source of current stock.

Review notes
- This is the main behavioral change: inventory is no longer just a code constant.
- Fixed weights are currently:
  대: 0.32kg
  중: 0.24kg
- Current implementation assumes known product/size/quality combinations. That is acceptable for the current apple-only MVP.
- Later improvement: move package_unit, sales_channel, and default price metadata into DB too, so SAMPLE_PRODUCTS can disappear.

10. Known Notes

- Test robot events were inserted during verification.
  Pro DB received:
  대/상 +0.32kg
  중/중 +0.24kg
  Free local DB received:
  대/상 +0.64kg

- PowerShell command-line Korean JSON can become mojibake depending on encoding.
  TurtleBot Python code saved as UTF-8 should send normal Korean strings correctly.
  If shell testing is needed, Unicode escapes can be used:
  {"size_class":"\ub300","quality_grade":"\uc0c1"}

- Current robot payload is intentionally minimal:
  size_class
  quality_grade
  product_name is optional and defaults to 사과.
