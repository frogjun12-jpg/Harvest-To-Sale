Fruits Local LLM RAG MVP - 최근 변경분 코드리뷰 요약
대상 커밋: caa432a Add sales marketplace and price refresh workflow
작성 범위: 새로 생긴 파일/기능과 삭제되거나 대체된 흐름만 정리한다.
제외 범위: 기존 파일 전체 함수 분석, 변경 없는 기존 로직 설명

--------------------------------------------------------------------------------
1-1. app/api/sales.py
--------------------------------------------------------------------------------

새로 생긴 역할
- FastAPI에 /sales 하위 API를 추가했다.
- 판매 재고 조회, 판매 초안 생성/수정/승인/등록, 쇼핑몰 등록상품 조회, 주문 생성, 알림 조회/읽음 처리를 담당한다.

새로 생긴 모델
- ProductInventory
  - MariaDB 판매 재고 요약 응답 모델이다.
  - size_class, grade, 기준 재고, 등록 재고, 판매 완료 수량, 추가 등록 가능 재고를 포함한다.
- SalesDraftRequest
  - 판매 초안 생성/수정 요청 모델이다.
  - product_name, size_class, grade, quantity_kg, estimated_unit_weight_kg, price_per_kg 등을 검증한다.
- SalesDraft
  - 판매 초안 응답 모델이다.
- SalesListing
  - 쇼핑몰 등록상품 응답 모델이다.
  - original_quantity_kg와 quantity_kg를 분리해 최초 등록 수량과 남은 판매 수량을 구분한다.
- SalesOrderRequest / SalesOrder
  - 쇼핑몰 주문 생성 요청/응답 모델이다.
- Notification
  - 농부 알림 응답 모델이다.

새로 생긴 엔드포인트
- GET /sales/products
  - 현재 판매 가능한 재고 목록을 반환한다.
- GET /sales/drafts
  - 판매 초안 목록을 반환한다.
- POST /sales/drafts
  - 판매 초안을 생성한다.
- PUT /sales/drafts/{draft_id}
  - 판매 초안을 수정한다.
- POST /sales/drafts/{draft_id}/approve
  - 판매 초안을 승인 상태로 바꾼다.
- POST /sales/drafts/{draft_id}/register
  - 승인 또는 초안 상태의 판매 초안을 쇼핑몰 상품으로 등록한다.
- GET /sales/listings
  - 쇼핑몰 등록상품 목록을 반환한다.
- GET /sales/orders
  - 주문 목록을 반환한다.
- POST /sales/listings/{listing_id}/orders
  - 쇼핑몰 주문을 생성하고 상품 남은 수량을 줄인다.
- GET /sales/notifications
  - 알림 목록을 반환한다.
- POST /sales/notifications/{notification_id}/read
  - 알림을 읽음 처리한다.

리뷰 포인트
- 모든 예외를 Exception으로 받아 400으로 바꾸고 있다. DB 장애, 검증 실패, 프로그래밍 오류가 같은 방식으로 노출될 수 있다.
- 현재 인증/권한이 없다. 내부 MVP라면 괜찮지만 외부 접속 테스트 시 판매 등록/주문 API가 열려 있다.
- 주문 생성은 DB 함수에서 수량 차감까지 처리하므로 동시 주문 경쟁 조건을 DB 트랜잭션 관점에서 계속 확인해야 한다.

1-2. app/db/sales.py
--------------------------------------------------------------------------------

새로 생긴 역할
- 판매 재고, 판매 초안, 쇼핑몰 등록상품, 주문, 알림의 MariaDB CRUD를 담당한다.
- 현재 실제 로봇 연동 전 단계라 SAMPLE_PRODUCTS 더미 재고를 기준 데이터로 사용한다.

새로 생긴 주요 데이터
- SAMPLE_PRODUCTS
  - 사과를 크기 대/중, 상품성 상/중/하 조합으로 관리한다.
  - 각 조합에 기준 재고, 예약 재고, kg당 권장가, 판매 단위, 추정 개당 중량을 둔다.

새로 생긴 주요 함수
- normalize_quality_grade()
  - 특/보통/중 같은 표현을 상/중/하 체계로 정규화한다.
- normalize_size_class()
  - 대과/중과/특대 같은 표현을 대/중 체계로 정규화한다.
- get_product_template()
  - product_name, size_class, grade에 맞는 SAMPLE_PRODUCTS 항목을 찾는다.
- get_listing_commitments()
  - 등록상품과 주문을 합산해 listed_kg, sold_kg, remaining_listing_kg를 계산한다.
- get_available_kg()
  - 기준 재고에서 이미 쇼핑몰에 등록한 총량을 빼 추가 등록 가능 재고를 계산한다.
- validate_new_listing_quantity()
  - 새 등록 수량이 추가 등록 가능 재고를 넘지 않는지 검사한다.
- extract_quantity_kg()
  - 챗봇 문장에서 100kg, 100키로 같은 수량을 뽑는다.
- extract_grade()
  - 챗봇 문장에서 상/중/하 품질 등급을 뽑는다.
- extract_size_class()
  - 챗봇 문장에서 대과/중과 크기를 뽑는다.
- list_products()
  - 기준 재고와 쇼핑몰 등록/판매 현황을 합쳐 API용 재고 목록을 만든다.
- parse_sales_registration_request()
  - 챗봇 질문과 최근 대화 문맥에서 판매 등록 요청을 파싱한다.
- register_listing_from_chat_request()
  - 챗봇 판매 요청을 바로 쇼핑몰 등록상품으로 만든다.
- create_notification()
  - 판매 등록, 주문 접수 같은 이벤트 알림을 만든다.
- create_draft(), update_draft(), approve_draft(), register_draft()
  - 판매 초안 워크플로우를 관리한다.
- place_order()
  - 주문 생성, 남은 판매 수량 차감, 품절 처리, 알림 생성을 한 흐름에서 처리한다.

리뷰 포인트
- SAMPLE_PRODUCTS는 코드 상수라 운영 중 재고 변경이 영속적으로 관리되지 않는다. 로봇팔/분류 모델 연동 시 별도 inventory 테이블로 옮기는 것이 좋다.
- parse_sales_registration_request()는 정규식/키워드 기반이라 문장이 애매하면 다른 등급 또는 크기로 잘못 등록할 수 있다.
- place_order()는 트랜잭션을 사용하지만, SELECT 후 UPDATE 흐름의 동시성 안정성을 실제 MariaDB 격리 수준에서 테스트해야 한다.
- create_draft()와 register_listing_from_chat_request() 모두 수량 검증을 하지만, 더미 기준 재고라 실제 수확 재고와 차이가 생길 수 있다.

1-3. app/api/prices.py
--------------------------------------------------------------------------------

새로 생긴 역할
- /prices/refresh 엔드포인트를 추가했다.
- GET은 이 주소가 실행 페이지가 아니라 POST 실행용 API라는 설명을 반환한다.
- POST는 시세 크롤링, Chronos 예측 문서 생성, RAG 재임베딩을 실행한다.

새로 생긴 엔드포인트
- GET /prices/refresh
  - 사용 방법 안내용 응답이다.
- POST /prices/refresh
  - 가격 데이터 갱신과 RAG 문서 재생성을 실행한다.

리뷰 포인트
- POST 실행 시간이 길 수 있다. Streamlit에서 호출할 때 timeout을 길게 잡았지만, 운영에서는 백그라운드 작업 큐가 더 적합하다.
- 외부 사이트 구조가 바뀌면 크롤러가 실패할 수 있다.

1-4. app/prices/garak_crawler.py
--------------------------------------------------------------------------------

새로 생긴 역할
- garakprice.com에서 사과 가격 데이터를 월 단위로 가져와 CSV로 저장한다.

새로 생긴 주요 함수
- parse_int()
  - 가격 문자열에서 숫자만 추출한다.
- parse_date()
  - 날짜 문자열을 pandas Timestamp로 변환한다.
- extract_kg_unit()
  - 단위 문자열에서 kg 단위를 추출한다.
- fetch_month()
  - 특정 연월의 HTML을 요청한다.
- parse_price_rows()
  - HTML 테이블 또는 텍스트 라인에서 가격 행을 파싱한다.
- crawl_garak_apple_prices()
  - 여러 달의 가격 데이터를 합쳐 DataFrame으로 만든다.
- save_latest_prices()
  - DataFrame을 UTF-8 BOM CSV로 저장한다.
- crawl_and_save_latest_prices()
  - 크롤링부터 저장까지 실행하는 진입 함수다.

리뷰 포인트
- HTML 구조 의존도가 높다. 사이트 마크업 변경 시 빈 데이터가 나올 수 있다.
- DEFAULT_YEAR가 현재 연도 기준이라 과거 데이터를 다시 만들 때 환경변수 설정이 필요하다.
- 네트워크 실패, 차단, 응답 지연을 재시도하지 않는다.

새로 생긴 역할

- 쇼핑몰 등록상품을 고객 화면처럼 보여주고 주문 테스트를 생성한다.

새로 생긴 주요 함수
- api_get(), api_post()
  - FastAPI sales API 호출 래퍼다.
- format_won()
  - 금액에 쉼표와 원 단위를 붙인다.
- item_name()
  - 상품명, 크기, 등급을 합친 표시명을 만든다.

- render_summary()
  - 상품 개수, 남은 수량, 등록 금액 요약 카드를 렌더링한다.

새로 생긴 화면 흐름
- 상단 마켓 헤더와 사과 이미지 영역
- 전체/대과/중과 버튼 필터
- 판매 중인 상품 수, 남은 판매 수량, 등록 금액 요약

- 고객 화면인데 인증이 없으므로 외부 접속 테스트 때 누구나 주문 테스트를 누를 수 있다.
- 필터가 크기 기준만 있고 등급 필터는 제거되어 현재 요구에는 맞지만 상품이 많아지면 검색/정렬이 필요해질 수 있다.

리뷰 포인트

- original_quantity_kg 복원 UPDATE는 기존 주문 데이터 상태에 따라 결과가 달라질 수 있다.

2-3. app/api/chat.py
--------------------------------------------------------------------------------

새로 생긴 기능
- 재고 질문 직접 응답
  - is_inventory_question(), build_inventory_answer()로 재고 질문은 RAG/LLM 대신 MariaDB 판매 재고 기준으로 답변한다.
- 크기/등급 파싱
  - get_requested_size_class(), get_requested_grade()가 대과/중과, 상/중/하 질문을 구분한다.
- 챗봇 판매 등록
  - register_listing_from_chat_request()를 호출해 "중과 하 20kg 쇼핑몰에 올려줘" 같은 요청을 바로 등록한다.
- 실시간 재고 context
  - build_live_inventory_context()를 source chunk처럼 응답에 붙인다.

리뷰 포인트
- 재고 질문이면 RAG/LLM을 건너뛰므로 답변은 빨라지지만 표현 다양성은 줄어든다.
- 판매 등록 파싱이 실패하면 일반 RAG 답변으로 넘어가거나, 수량 초과만 명시적으로 오류 응답한다.
- "중"이 크기인지 등급인지 헷갈리는 문장에 대한 예외처리가 계속 중요하다.

2-4. app/ui/streamlit_app.py
--------------------------------------------------------------------------------

새로 생긴 기능
- 단일 챗봇 화면에서 관리자 대시보드형 Streamlit 앱으로 확장되었다.
- 사이드바 버튼 메뉴
  - 대시보드, 챗봇, 판매등록, 쇼핑몰 재고, 등록상품, 시세갱신, 알림 화면을 전환한다.
- 대시보드 카드
  - 총 재고, 쇼핑몰 등록 재고, 추가 등록 가능 재고, 최근 주문 지표를 보여준다.
- 챗봇 패널
  - 질문 입력과 최신 답변, 최근 대화를 분리해 보여준다.
- 판매등록 화면
  - 재고 선택, 수량/가격/판매 단위 입력, 초안 생성, 승인, 쇼핑몰 등록 흐름을 제공한다.
- 쇼핑몰 재고 화면
  - 쇼핑몰에 노출된 남은 판매 수량을 간단히 보여준다.
- 등록상품 화면
  - 등록상품 전체 목록을 보여준다.
- 시세갱신 화면
  - 버튼으로 /prices/refresh POST를 호출한다.
- 알림 화면
  - 주문/등록 알림을 확인하고 읽음 처리한다.
- 실시간 알림 사이드바
  - 일정 주기로 알림을 다시 확인한다.

리뷰 포인트
- 한 파일에 화면이 많이 모여 있어 앞으로 유지보수가 어려워질 수 있다. 페이지별 함수 분리가 필요해질 가능성이 높다.
- Streamlit UI에서 실제 판매 등록과 주문 데이터 변경이 일어나므로 버튼 라벨과 확인 절차가 중요하다.
- 현재 CSS와 HTML이 많아 Streamlit 기본 컴포넌트와 충돌할 가능성이 있다.

4. 새로 생긴 변경분에서 우선 확인할 사항
================================================================================

1. 판매 등록 수량 검증
- 쇼핑몰 등록 수량이 기준 재고보다 커질 수 없는지 확인해야 한다.
- 같은 상품을 여러 번 등록했을 때 추가 등록 가능 재고가 정확히 줄어드는지 확인해야 한다.

2. 주문 생성 시 남은 판매 수량 차감
- 주문 후 sales_listings.quantity_kg가 줄어드는지 확인해야 한다.
- 0kg이 되면 status가 sold_out으로 바뀌는지 확인해야 한다.

3. 챗봇 판매 등록 파싱
- "보통 등급", "중 등급", "중과"가 섞인 문장에서 크기와 품질을 잘못 해석하지 않는지 확인해야 한다.

4. 외부 접속 시 API 노출
- 0.0.0.0으로 실행하면 /sales와 /prices가 같은 네트워크에 열린다.
- 교수님 데모용이면 괜찮지만 실제 운영에는 인증/승인 절차가 필요하다.

5. 크롤러 안정성
- garakprice.com HTML 구조가 바뀌면 parse_price_rows()가 실패할 수 있다.
- 실패 시 기존 RAG 문서를 덮어쓸지 말지 정책이 필요하다.

6. 데이터 소스 일관성
- app/db/sales.py의 SAMPLE_PRODUCTS와 rag_docs/fruit_price_inventory_sales.md의 수치가 다르면 챗봇 답변이 서로 충돌할 수 있다.
